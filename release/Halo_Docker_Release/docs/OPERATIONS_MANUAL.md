## 1. 登录与基础设置

### 首次登录

1. 打开浏览器访问 http://localhost:8000/admin
2. 输入默认账号：
   - 用户名：`admin`
   - 密码：`admin123`
3. 点击「登录」按钮

### 修改管理员密码

在 `docker-compose.yml` 中修改环境变量：

```yaml
environment:
  - HALO_ADMIN_PASSWORD=你的新密码
```

然后重启容器：

```bash
cd E:\Docker\Halo_Docker
docker compose up -d
```

> ⚠️ **注意：** 修改密码后，已登录的会话仍有效（Token 24 小时过期）。如需立即生效，清除浏览器 localStorage 中的 `admin_token`。

### 登录状态说明

- Token 存储在浏览器 `localStorage` 中
- 默认有效期：**24 小时**
- 过期后需重新登录
- 关闭浏览器不会退出登录

## 3. 文章管理完整指南

### 3.1 查看文章列表

点击左侧导航「📝 文章管理」，进入文章列表页面。

**列表字段说明：**

| 字段 | 说明 |
|------|------|
| 标题 | 可点击跳转到前台预览 |
| 状态 | `已发布`（绿色）/ `草稿`（黄色） |
| 分类 | 所属分类名称 |
| 标签 | 文章标签列表 |
| 浏览 | 累计浏览次数 |
| 发布时间 | 首次发布时间 |
| 操作 | 编辑 / 删除 |

**功能：**
- 🔍 **搜索** — 输入关键词后按回车搜索
- 📄 **翻页** — 底部分页按钮
- ➕ **写文章** — 跳转到编辑器

### 3.2 写文章（Markdown 编辑器）

点击「✏️ 写文章」或文章列表中的「编辑」按钮，进入编辑器。

#### 编辑器界面

```
┌─────────────────────────────────────────────────────┐
│  ← 返回列表  💾 保存  📤 发布                        │
├─────────────────────────────────────────────────────┤
│  标题: [_________________________________]          │
│  Slug: [___________]   分类: [选择分类 ▼]           │
│  摘要: [_________________________________]          │
├──────────────────────┬──────────────────────────────│
│  Markdown 编辑器     │  实时预览                     │
│                      │                              │
│  # 标题              │  标题                         │
│  正文内容...         │  正文内容...                  │
│                      │                              │
│  **粗体**            │  粗体                         │
│  `代码`              │  代码                         │
└──────────────────────┴──────────────────────────────┘
```

#### 各字段详解

| 字段 | 必填 | 说明 |
|------|------|------|
| **标题** | ✅ | 文章标题，显示在首页和详情页 |
| **Slug** | ❌ | URL 别名，如不填会自动从标题生成 |
| **分类** | ❌ | 下拉选择已有分类 |
| **摘要** | ❌ | 文章简介，显示在首页卡片中 |
| **内容** | ❌ | Markdown 格式正文 |

#### 操作按钮

| 按钮 | 功能 |
|------|------|
| 💾 **保存** | 保存为草稿（`status=draft`） |
| 📤 **发布** | 发布文章（`status=published`），同时设置发布时间 |

#### Markdown 语法支持

```markdown
# 一级标题
## 二级标题
### 三级标题

**粗体文字**
*斜体文字*
`行内代码`

- 无序列表
1. 有序列表

> 引用块

| 表头1 | 表头2 |
|-------|-------|
| 内容1 | 内容2 |

```python
# 代码块（支持语法高亮）
def hello():
    print("Hello World!")
```

[链接文字](https://example.com)
![图片描述](/uploads/image.jpg)
```

### 3.3 文章状态流转

```
                    💾 保存
    ┌──────────────────────────────────────┐
    │                                      ▼
    │                               ┌──────────┐
    │                               │  草稿    │
    │                               │ (draft)  │
    │                               └────┬─────┘
    │                                    │
    │                           📤 发布  │
    │                                    ▼
    │                               ┌──────────┐
    └──────────────────────────────│  已发布  │
              重新编辑保存          │(published)│
                                   └──────────┘
```

- **草稿**：不会在前台显示，不计入已发布统计
- **已发布**：在前台可见，自动设置 `published_at` 时间戳
- **重新编辑**：已发布文章修改后仍保持 `published` 状态

### 3.4 文章删除

1. 在文章列表中点击「删除」按钮
2. 弹出确认对话框
3. 确认后**永久删除**文章及其关联的评论

> ⚠️ **警告：** 删除操作不可恢复！建议先备份重要内容。

### 3.5 Slug 规则

- Slug 是文章的 URL 标识
- 只能包含小写字母、数字和连字符
- 全局唯一，不可重复
- 示例：`my-first-blog-post`

**自动生成功能：**
- 如果留空，系统会从标题自动生成
- 中文标题会转为拼音或删除
- 建议手动设置简短有意义的英文 slug

### 3.6 封面图片

封面图片通过 API 设置，支持：
- 上传到 `/admin/api/upload` 获取 URL
- 使用外部图片链接
- 留空则不显示封面

## 4. 分类管理

### 4.1 查看分类列表

点击左侧导航「📂 分类管理」。

**列表字段：**

| 字段 | 说明 |
|------|------|
| 名称 | 分类显示名称 |
| Slug | URL 别名 |
| 描述 | 分类简介 |
| 文章数 | 该分类下的文章数量 |

### 4.2 创建新分类

1. 点击「➕ 新增分类」按钮
2. 填写表单：

| 字段 | 必填 | 说明 |
|------|------|------|
| 名称 | ✅ | 分类名称（如：技术、生活、随笔） |
| Slug | ❌ | URL 别名，不填自动生成 |
| 描述 | ❌ | 分类简介文字 |

3. 点击「保存」按钮

### 4.3 分类与文章的关系

- 每篇文章最多属于 **1 个分类**
- 分类下有文章时，删除分类**不会**删除文章（文章的 `category_id` 变为 `null`）
- 前台首页侧边栏显示分类及文章计数

### 4.4 分类设计建议

| 分类 | 适用场景 |
|------|---------|
| 技术 | 编程、开发、运维相关 |
| 生活 | 日常记录、旅行、美食 |
| 随笔 | 感想、思考、读书笔记 |
| 项目 | 项目总结、作品展示 |

## 5. 标签管理

### 5.1 创建标签

1. 点击左侧导航「🏷️ 标签管理」
2. 填写「标签名称」和「标签 Slug」
3. 点击「➕ 添加」按钮

### 5.2 标签与分类的区别

| 特性 | 分类 | 标签 |
|------|------|------|
| 数量限制 | 每篇最多 1 个 | 每篇可多个 |
| 层级 | 可扩展为树状 | 扁平结构 |
| 用途 | 文章归类 | 关键词标记 |
| 显示位置 | 文章元信息 + 侧边栏 | 文章元信息 + 标签云 |

### 5.3 标签使用建议

- 每篇文章建议 **2-5 个标签**
- 标签名称简短（2-4 个字）
- 避免创建太多相似标签
- 常用标签示例：`Python`、`Docker`、`教程`、`笔记`、`Bug修复`

## 6. 评论管理

### 6.1 评论审核流程

```
访客提交评论 → 默认「待审核」→ 管理员审核 → 通过后前台显示
                                    │
                                    └→ 删除（垃圾评论）
```

### 6.2 查看评论

点击左侧导航「💬 评论管理」。

**列表字段：**

| 字段 | 说明 |
|------|------|
| 作者 | 评论者昵称 + 邮箱 |
| 内容 | 评论内容摘要（前 80 字） |
| 状态 | `待审核`（橙色）/ `已通过`（绿色） |
| 时间 | 提交时间 |
| 操作 | ✅ 通过 / 删除 |

### 6.3 审核评论

- 点击「✅ 通过」按钮，评论状态变为 `is_approved=True`
- 通过后立即在前台对应文章页面显示
- 支持嵌套回复（回复会显示在父评论下方）

### 6.4 评论数据

每条评论包含：

| 字段 | 说明 |
|------|------|
| `author_name` | 昵称（必填） |
| `author_email` | 邮箱（可选，不公开显示） |
| `author_website` | 网站（可选，显示为链接） |
| `content` | 评论内容（最多 2000 字） |
| `ip_address` | 提交者 IP（自动记录） |
| `parent_id` | 父评论 ID（嵌套回复） |

### 6.5 前台评论表单

访客在文章详情页底部可以看到评论表单：

- **昵称**（必填）
- **邮箱**（可选）
- **网站**（可选）
- **评论内容**（必填）

提交后显示提示：「评论已提交，等待审核通过后显示」

## 8. 系统参数配置详解

### 8.1 博客基本信息

通过 `docker-compose.yml` 环境变量配置：

```yaml
environment:
  # 博客标题 — 显示在浏览器标签页和页面顶部
  - HALO_BLOG_TITLE=我的技术博客

  # 副标题 — 显示在首页 Hero 区域
  - HALO_BLOG_SUBTITLE=记录 Python 与 Docker 学习之路

  # 页脚文字 — 显示在每个页面底部
  - HALO_BLOG_FOOTER=© 2026 我的博客 | 保留所有权利
```

### 8.2 文章显示设置

```yaml
environment:
  # 每页显示文章数（默认 10）
  - HALO_POSTS_PER_PAGE=15
```

**生效页面：**
- 首页文章列表
- 分类筛选结果
- 标签筛选结果
- 搜索结果

### 8.3 安全设置

```yaml
environment:
  # JWT 签名密钥（生产环境必须修改！）
  - HALO_SECRET_KEY=your-random-secret-key-at-least-32-chars

  # 管理员密码（生产环境必须修改！）
  - HALO_ADMIN_PASSWORD=YourStrongPassword123!
```

### 8.4 调试模式

```yaml
environment:
  # 开启调试模式（显示 SQL 查询日志）
  - HALO_DEBUG=true
```

> ⚠️ **生产环境不要开启 `HALO_DEBUG=true`**，会暴露 SQL 查询细节。

### 8.5 上传设置

```yaml
environment:
  # 上传文件目录（容器内路径）
  - HALO_UPLOAD_DIR=uploads

  # 最大上传大小（字节）
  # 10MB = 10485760
  # 20MB = 20971520
  # 50MB = 52428800
  - HALO_MAX_UPLOAD_SIZE=10485760
```

### 8.6 服务器设置

```yaml
environment:
  # 监听地址（0.0.0.0 = 所有网卡）
  - HALO_HOST=0.0.0.0

  # 监听端口
  - HALO_PORT=8000
```

**修改外部端口映射：**

```yaml
ports:
  - "9000:8000"   # 外部用 9000 端口访问
```

### 8.7 数据库设置

```yaml
environment:
  # SQLite（默认，轻量级）
  - HALO_DATABASE_URL=sqlite+aiosqlite:///./data/blog.db

  # PostgreSQL（高级选项）
  - HALO_DATABASE_URL=postgresql+asyncpg://user:pass@host:5432/dbname
```

### 8.8 配置修改后的生效方式

| 配置类型 | 生效方式 |
|----------|---------|
| 环境变量 | `docker compose restart` |
| 修改源代码 | `docker compose up -d --build` |
| 修改静态文件 | `docker compose up -d --build` |
| 修改模板文件 | `docker compose up -d --build` |

## 9. 前端页面注意事项

### 9.1 页面路由

| 路径 | 页面 | 说明 |
|------|------|------|
| `/` | 首页 | 文章列表 + 侧边栏 |
| `/post/{slug}` | 文章详情 | 全文 + 评论区 |
| `/categories` | 分类页 | 分类卡片网格 |
| `/archives` | 归档页 | 按年月分组 |
| `/links` | 友链页 | 友情链接列表 |
| `/admin` | 管理后台 | SPA 单页应用 |

### 9.2 首页功能

**文章卡片显示内容：**
- 封面图片（如有）
- 分类名称
- 发布日期
- 浏览次数 👁
- 评论数 💬
- 标题（置顶文章带 📌 标记）
- 摘要文字
- 标签列表

**侧边栏显示：**
- 分类列表（含文章计数）
- 标签云（含文章计数）

**筛选功能：**
- 点击分类 → 按分类筛选
- 点击标签 → 按标签筛选
- URL 参数：`/?category=python` 或 `/?tag=docker`

### 9.3 文章详情页

**显示内容：**
- 分类 + 发布日期 + 浏览数
- 文章标题
- 标签列表
- Markdown 渲染后的正文
- 评论区（如允许评论）

**代码高亮：**
- 使用 highlight.js 渲染
- 支持 180+ 种编程语言
- GitHub 风格样式

**评论区：**
- 显示已审核的评论
- 支持嵌套回复
- 评论表单：昵称（必填）、邮箱（可选）、网站（可选）、内容（必填）

### 9.4 前端注意事项

#### ⚠️ 不要使用 Markdown 表格

由于前端使用客户端渲染，Markdown 表格可能无法正确显示。建议使用列表代替。

#### ⚠️ 图片处理

- 上传图片通过 `/admin/api/upload` 接口
- 图片自动存储在 `uploads/` 目录
- 在 Markdown 中引用：`![描述](/uploads/filename.jpg)`
- 建议图片宽度不超过 800px

#### ⚠️ 代码块

- 必须指定语言标识符才能高亮
- 正确：` ```python `
- 错误：` ``` `（无语言标识）

#### ⚠️ 链接

- 内部链接：使用相对路径 `/post/hello-world`
- 外部链接：使用完整 URL `https://example.com`
- 外部链接默认在新窗口打开

#### ⚠️ 移动端适配

- 响应式设计，自动适配手机屏幕
- 导航栏在移动端折叠为汉堡菜单
- 侧边栏在移动端显示在文章列表上方
- 评论表单在移动端单列显示

#### ⚠️ SEO 注意事项

- 文章 slug 应简短有意义
- 标题不要超过 60 个字符
- 摘要控制在 150-200 字
- 图片添加 alt 描述
- 使用语义化的 Markdown 标题层级

### 9.5 前台页面刷新

- 前台页面使用 **客户端 JavaScript 渲染**
- 每次访问页面会调用 `/api` 接口获取最新数据
- **无需手动刷新缓存**
- 修改文章后立即在前台可见

### 9.6 浏览器兼容性

支持所有现代浏览器：
- ✅ Chrome 90+
- ✅ Firefox 90+
- ✅ Safari 14+
- ✅ Edge 90+
- ❌ IE 11（不支持）

## 10. 运维与日常维护

### 10.1 日常操作清单

| 频率 | 操作 |
|------|------|
| 每日 | 审核新评论 |
| 每周 | 检查容器状态、查看日志 |
| 每月 | 备份数据库、清理无用上传文件 |
| 每季度 | 更新密码、检查安全设置 |

### 10.2 数据备份

```bash
# 备份数据库（最重要的！）
copy E:\Docker\Halo_Docker\data\blog.db E:\Backup\blog_%date:~0,10%.db

# 备份上传文件
xcopy E:\Docker\Halo_Docker\uploads E:\Backup\uploads\ /E /I

# 完整备份
xcopy E:\Docker\Halo_Docker E:\Backup\Halo_Docker_backup\ /E /I
```

### 10.3 数据库恢复

```bash
# 停止容器
docker compose down

# 替换数据库文件
copy E:\Backup\blog.db E:\Docker\Halo_Docker\data\blog.db

# 启动容器
docker compose up -d
```

### 10.4 日志查看

```bash
# 查看最近日志
docker logs halo-blog

# 实时跟踪日志
docker logs -f halo-blog

# 查看最近 50 行
docker logs --tail 50 halo-blog
```

### 10.5 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 页面空白 | JavaScript 错误 | 打开浏览器控制台查看错误 |
| 评论不显示 | 未审核 | 后台审核评论 |
| 图片不显示 | 路径错误 | 检查 uploads 目录权限 |
| 登录失败 | 密码错误 | 修改环境变量重置密码 |
| 容器重启 | 配置错误 | 查看 `docker logs` |

### 10.6 性能优化建议

- **图片压缩** — 上传前压缩图片（推荐 < 500KB）
- **定期清理** — 删除未使用的上传文件
- **评论审核** — 及时清理垃圾评论
- **分类合理** — 不要创建过多分类（建议 5-10 个）
- **标签精简** — 定期合并相似标签
