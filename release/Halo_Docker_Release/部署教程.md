# Halo Blog 部署教程（小白版）

> 本教程面向零基础用户，一步步教你部署自己的博客网站。

---

## 一、准备工作

### 1.1 什么是 Docker？

Docker 就像一个"虚拟环境"，把博客程序和它需要的所有东西打包在一起。
你不需要安装 Python、数据库这些复杂的东西，只需要安装 Docker 就行。

### 1.2 安装 Docker Desktop

1. **下载 Docker Desktop**
   - 官网下载：https://www.docker.com/products/docker-desktop/
   - 选择 Windows 版本下载

2. **安装 Docker Desktop**
   - 双击下载的安装包
   - 一路点击「下一步」直到安装完成
   - 安装过程中可能会要求重启电脑，选择「是」

3. **启动 Docker Desktop**
   - 双击桌面的 Docker Desktop 图标
   - 等待右下角的鲸鱼图标变成**绿色**
   - 首次启动可能需要 1-2 分钟

4. **验证安装成功**
   - 右键点击开始菜单 → 选择「终端」或「PowerShell」
   - 输入以下命令并回车：
   ```
   docker version
   ```
   - 如果显示版本号（如 `Version: 29.x.x`），说明安装成功

> ⚠️ **系统要求：** Windows 10/11 64位，需要开启虚拟化（一般默认已开启）

## 二、部署博客

### 2.1 获取文件

将 `Halo_Docker_Release` 文件夹复制到你电脑的任意位置，比如：
- `D:\HaloBlog`
- `E:\MyBlog`
- 桌面上也行

### 2.2 一键启动

1. 打开 `Halo_Docker_Release` 文件夹
2. **双击运行 `一键启动.bat`**
3. 等待构建完成（首次需要 2-3 分钟下载依赖）
4. 浏览器会自动打开管理后台

### 2.3 首次登录

浏览器打开后会看到登录页面：

```
用户名：admin
密码：admin123
```

点击「登录」即可进入管理后台。

> ⚠️ **重要：** 首次登录后请立即修改密码！

## 三、基本使用

### 3.1 发表第一篇文章

1. 点击左侧「✏️ 写文章」
2. 填写标题（如：Hello World）
3. 在编辑区写内容（支持 Markdown 语法）
4. 点击「📤 发布」按钮
5. 访问 http://localhost:8000 查看效果

### 3.2 简单的 Markdown 语法

```markdown
# 一级标题

## 二级标题

这是一段普通文字。

**这是粗体**

*这是斜体*

- 无序列表项1
- 无序列表项2

1. 有序列表1
2. 有序列表2

> 这是一段引用

[这是一个链接](https://example.com)

```代码块```
```

### 3.3 创建分类

1. 点击左侧「📂 分类管理」
2. 点击「➕ 新增分类」
3. 填写名称（如：技术、生活、随笔）
4. 点击「保存」

### 3.4 添加标签

1. 点击左侧「🏷️ 标签管理」
2. 输入标签名称（如：Python、Docker）
3. 点击「➕ 添加」

### 3.5 审核评论

访客提交的评论需要你审核后才能显示：

1. 点击左侧「💬 评论管理」
2. 看到待审核的评论
3. 点击「✅ 通过」让评论显示
4. 或者点击「删除」移除垃圾评论

### 3.6 添加友情链接

1. 点击左侧「🔗 友链管理」
2. 点击「➕ 新增友链」
3. 填写网站名称和网址
4. 点击「保存」

## 五、常用操作

### 5.1 启动博客

双击运行 `一键启动.bat`

### 5.2 停止博客

双击运行 `一键停止.bat`

### 5.3 重启博客

先运行「一键停止」，再运行「一键启动」

### 5.4 查看博客状态

右键开始菜单 → 打开「终端」→ 输入：

```
docker ps --filter "name=halo-blog"
```

如果显示 `Up` 开头的文字，说明正在运行。

### 5.5 查看日志（出错时使用）

在终端中输入：

```
docker logs halo-blog
```

## 六、修改博客设置

### 6.1 修改博客标题/副标题

1. 用记事本打开 `docker-compose.yml` 文件
2. 找到这些行并修改：

```yaml
environment:
  - HALO_BLOG_TITLE=我的个人博客
  - HALO_BLOG_SUBTITLE=记录生活与技术
```

3. 保存文件
4. 重新运行「一键启动」

### 6.2 修改管理员密码

同样在 `docker-compose.yml` 中：

```yaml
environment:
  - HALO_ADMIN_PASSWORD=你的新密码
```

保存后重新启动。

### 6.3 修改每页显示文章数

```yaml
environment:
  - HALO_POSTS_PER_PAGE=15
```

## 七、数据备份

### 7.1 重要数据在哪里？

你的博客数据都在 `Halo_Docker_Release` 文件夹里：

```
Halo_Docker_Release/
├── data/           ← 数据库（所有文章、评论都在这里！）
│   └── blog.db
└── uploads/        ← 上传的图片
```

### 7.2 如何备份？

**最简单的方法：** 直接复制整个 `Halo_Docker_Release` 文件夹！

或者只备份重要数据：

```
Halo_Docker_Release/
├── data/blog.db    ← 复制这个文件（文章和评论）
└── uploads/        ← 复制这个文件夹（图片）
```

### 7.3 如何恢复？

1. 停止博客（运行「一键停止」）
2. 将备份的 `blog.db` 放回 `data/` 目录
3. 将备份的 `uploads/` 放回原位置
4. 启动博客（运行「一键启动」）

## 八、常见问题

### Q: 启动时提示"Docker 未安装"

**A:** 需要先安装 Docker Desktop，见[第一步](#一准备工作)。

### Q: 启动时提示"Docker 未启动"

**A:** 双击桌面的 Docker Desktop 图标，等右下角鲸鱼图标变绿后重试。

### Q: 启动时提示"端口被占用"

**A:** 8000 端口被其他程序占用了。解决方法：

1. 用记事本打开 `docker-compose.yml`
2. 修改端口映射：
```yaml
ports:
  - "9000:8000"    # 改成 9000
```
3. 保存后重新启动
4. 用 http://localhost:9000 访问

### Q: 浏览器打不开页面

**A:** 检查以下几点：
1. Docker Desktop 是否在运行（右下角鲸鱼图标是否绿色）
2. 是否等待构建完成（首次启动需要 2-3 分钟）
3. 尝试访问 http://127.0.0.1:8000

### Q: 忘记管理员密码怎么办？

**A:** 用记事本打开 `docker-compose.yml`，修改：
```yaml
- HALO_ADMIN_PASSWORD=新密码
```
保存后重新启动。

### Q: 如何让别人通过互联网访问？

**A:** 需要以下步骤：
1. 购买一个域名（如阿里云、腾讯云）
2. 将域名解析到你的公网 IP
3. 在路由器设置端口转发（8000 端口）
4. 或者使用内网穿透工具（如 frp、ngrok）

这部分比较复杂，建议搜索"内网穿透教程"了解详情。

### Q: 数据会丢失吗？

**A:** 只要不删除以下文件夹，数据就不会丢失：
- `data/` — 数据库
- `uploads/` — 上传的图片

「一键停止」只是停止运行，不会删除数据。
